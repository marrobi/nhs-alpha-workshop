## GDS Service Standard Evidence

### Goal
Map the repository's implementation evidence to the 14-point GDS Service Standard for NHS Alpha assessment readiness.

### Acceptance Criteria
- [ ] `docs/gds-assessment/` directory created
- [ ] Evidence report covers all 14 GDS Service Standard points
- [ ] Each point has: current status (Met / Partially Met / Not Met / Not Applicable), evidence summary, and gap analysis
- [ ] Point 5 (iterate and improve) references ADRs and iteration history
- [ ] Point 7 (use agile ways of working) references GitHub Issues and PRs
- [ ] Point 9 (test with real users) has placeholder for user research findings
- [ ] Point 10 (accessibility) references WCAG 2.2 AA audit results and axe-core output
- [ ] Point 12 (technology) maps to Azure UK South, Managed Identity, Key Vault, FastAPI, React
- [ ] Point 14 (operate a reliable service) references monitoring (Application Insights) and deployment runbook
